# 🏪 Departmental Store Management System

A console-based inventory management application written in **C** for a departmental store. Built as a **Term 2 Programming Project**, it allows store staff to manage products — adding, selling, searching, editing, deleting, and listing sold-out goods — all through a simple interactive menu.

## ✨ Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | **Add Product** | Register a new product with ID, name, price, and quantity. Validates input and checks for duplicate IDs. |
| 2 | **Sell Product** | Search by ID and reduce quantity by the sold amount. Prevents overselling. |
| 3 | **Search Product** | Look up a product by its ID and display full details. |
| 4 | **List Sold-Out Goods** | Display names of all products with zero quantity remaining. |
| 5 | **List Sold-Out Quantities** | Display full details of all products with zero quantity. |
| 6 | **Edit Product** | Update any field (ID, name, price, quantity) of an existing product. |
| 7 | **Delete Product** | Remove a product from the list by ID; shifts remaining items to fill the gap. |
| 8 | **Exit** | Cleanly exit the program. |

## 🛠️ How to Compile & Run

### Prerequisites
- A C compiler: **GCC** (MinGW on Windows) or any standard C99-compatible compiler.

### Compile

```bash
gcc project.c -o project
```

### Run

```bash
./project
```

> On Windows, you can also double-click **`project.exe`** directly.

## 🗂️ Data Model

Each product is stored in memory using the following struct:

```c
struct Product {
    int   id;                  // Unique product identifier
    char  product_name[50];    // Product name (up to 49 characters)
    float price;               // Price per unit
    int   quantity;            // Available stock quantity
};
```

The system supports a maximum of **100 products** at a time (`#define limit 100`).
